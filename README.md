## Overview

This project implements a **Logistics & Supply Chain Management System** for a **Bank Management System** using **SQL**. It focuses on managing the procurement, storage, distribution, and tracking of bank resources such as documents, cash logistics, equipment, and branch supplies.

## Features

* Branch and warehouse management
* Supplier and vendor tracking
* Inventory management (items, stock levels)
* Purchase orders and deliveries
* Shipment and distribution tracking
* Transaction logging and audit trails
* Role-based access (admin, manager, staff)

## Database Design

The system is built on a **relational database** with core tables including:

* `Branches`
* `Suppliers`
* `Products`
* `Inventory`
* `Orders`
* `Shipments`
* `Payments`

Relationships are enforced using **primary keys, foreign keys, and constraints** to ensure data integrity.

## Technologies

* SQL (MySQL / PostgreSQL / SQL Server)
* ER Diagrams & Normalized Schema

## Use Cases

* Track inventory across bank branches
* Manage supplier orders and deliveries
* Monitor logistics costs and payments
* Generate operational and audit reports

## How to Run

1. Create the database using the provided SQL scripts
2. Insert sample data (optional)
3. Run queries to manage and analyze logistics operations
